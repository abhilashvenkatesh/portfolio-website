## Overview

Quiet Technical Confidence meets Warm Minimalism. The portfolio communicates depth without noise — a senior engineer's work speaks in negative space. Warm parchment grounds the light theme; deep void anchors the dark. A single burnt-orange accent (`oklch(0.58 0.18 38)` light / `oklch(0.65 0.18 38)` dark — approximated as `#c0612b` in tokens) provides every interactive cue and decorative highlight. Two fonts carry the whole system: DM Sans for humanity, DM Mono for precision.

The grid is invisible but present — 60px squares behind hero sections, masked to a radial vignette so the structure implies itself without asserting.

## Colors

Two themes share one accent. The palette stays warm and neutral, with one deliberate hot spot.

**Light (Sand) — default theme:**
- **Primary (#0c0a09):** Near-black ink. Headings, max contrast.
- **Secondary (#78716c):** Warm stone. Muted text, metadata, nav labels.
- **Tertiary (#c0612b):** Burnt orange — "the accent." All interactions, links, CTA, decorative highlights. Source value is `oklch(0.58 0.18 38)`.
- **Neutral (#f5f0e8):** Warm parchment. Page background. Not white — never white.
- **Surface (#fdfaf4):** Slightly lighter parchment for card/input backgrounds.
- **Surface-alt (#ede8de):** Warm stone for borders, subtle fills, active nav, scrollbar.

**Dark (Midnight):**
- Background scales from `#080809` → `#0f0f11` → `#161618` for depth layering.
- Text inverts to `#e8e8ea`, muted to `#6b6b72`, headings to `#f0f0f2`.
- Accent brightens to `oklch(0.65 0.18 38)` — same hue, lifted lightness.
- No box shadows in dark mode — depth through background layering only.

Accent at 10–13% opacity forms `accent-dim` (hover fills, selection highlight). Accent at 30–38% opacity forms `accent-border` (input focus rings, pill hover borders). These two derived values appear everywhere interactive.

**Prototype color schemes (blog page exploration):** The blog page ships 5 swappable schemes for design exploration. These are not production variants — Sand is canonical.

| Scheme | Bg | Accent |
|---|---|---|
| Sand (default) | `#f5f0e8` | `oklch(0.58 0.18 38)` — burnt orange |
| Midnight | `#080809` | `oklch(0.72 0.16 195)` — teal |
| Obsidian | `#0c0c0e` | `oklch(0.75 0.18 290)` — violet |

Each dark scheme shares the same structural CSS variable names — only accent hue and background tone change. This demonstrates the system's portability.

## Typography

DM Sans + DM Mono. No other typefaces. The pairing is warmly geometric (Sans) and crisply utilitarian (Mono).

**DM Sans** — primary face for all prose, headings, and UI labels. Loaded at weights 300 (light), 400 (regular), 500 (medium), 600 (semibold). Italics at 300 only.

**DM Mono** — secondary face used exclusively for: logotype (`abhilash`), skill/tech tags, stat labels, nav resume button, footer links, chat suggestion pills, section metadata, company names on timeline, period badges, category headers on skill cards, inline code. Creates a "terminal" register that signals technical precision.

**Fraunces** — loaded on blog.html (serif, weights 400/600, italic 400). Currently unused in active components. Reserved for potential editorial contexts (pull-quotes, feature blog headers).

**Scale:**
- Hero display: `clamp(48px, 7vw, 88px)`, weight 600/300 split (300 for secondary lines), tracking `-0.03em`, line height `1.05`.
- Page h1: `clamp(32px, 5vw, 56px)`, weight 600, tracking `-0.03em`, line height `1.1`.
- Blog post h1: `clamp(28px, 4vw, 44px)`, weight 600, tracking `-0.03em`, line height `1.15`.
- Prose h2: `clamp(18px, 2.5vw, 24px)`, weight 600, tracking `-0.02em`, line height `1.25`.
- Card title: `20px`, weight 600, tracking `-0.02em`.
- Body lead (about/contact pages): `17px`, weight 400, line height `1.75`.
- Body large (hero sub): `clamp(16px, 2vw, 19px)`, weight 300, line height `1.7`.
- Body: `16px`, weight 400, line height `1.6`.
- Body small (chat, lists, timeline bullets): `15px`, weight 400, line height `1.7`.
- Prose body: `16px`, line height `1.85`.
- Labels (Mono): `11–13px` with `0.03–0.1em` letter spacing.
- Stat numbers: `28px`, weight 600, tracking `-0.02em`.

Tight negative tracking on large headings is load-bearing to the aesthetic — never loosen it.

## Layout

Single-column centered content with generous breathing room.

- **Max content width:** 1100px (global), with page-specific narrower containers: 860px (about), 720px (experience, blog post), 640px (contact), 780px (chat).
- **Page inline padding:** `clamp(20px, 5vw, 64px)` — fluid, never cramped on mobile.
- **Vertical rhythm:** 60/80px section padding. Hero: `120px` top, `80px` bottom. Page content: `60px` top.
- **Nav height:** 60px fixed, `backdrop-filter` blur over scrolled content at 90% opacity. `padding-top: 60px` on `<main>` compensates.
- **Footer:** minimal, `border-top`, `24px` block padding, flex row with logo + links.

Grid decorations appear only in hero sections: `60px × 60px` line grid, masked radially at `80% × 60%` ellipse so edges dissolve. Not interactive. Not repeated outside hero.

Responsive breakpoint at `640px` hides nav links — mobile shows logo and theme toggle only.

## Elevation & Depth

Light mode uses a single shadow token: `0 1px 3px rgba(0,0,0,0.06), 0 4px 16px rgba(0,0,0,0.04)`. Shallow and diffuse — lift without drama.

Dark mode removes shadows entirely. Depth is achieved through background layering (`bg` < `bg2` < `bg3`) and border opacity (`rgba(255,255,255,0.07)`).

Blog cards in filled style apply a stronger hover shadow: `0 8px 32px rgba(0,0,0,0.18)` — only used for that elevated card variant in light mode.

The hero glow is the only ambient effect: an `oklch(0.58 0.18 38 / 0.08)` ellipse, `500×400px`, blurred at `80px`, centered behind the heading. It is decorative atmosphere, not a UI primitive.

## Shapes

**Border radius scale:**
- `4px` (xs) — inline tags (Tag, AccentTag), inline code.
- `6px` (sm) — nav links, secondary buttons.
- `8px` (md) — theme toggle, code blocks, avatar (blog post), skill chip containers.
- `10px` (lg) — primary icon CTA, skill cards, contact icon container, chat avatar (`8px`).
- `12px` (card) — project cards, blog cards, contact cards, experience CTA, author card, chat message bubbles, "more posts" list items.
- `14px` (xl) — chat input container.
- `100px` (full) — suggestion pills, filter pills, period badges on timeline.

`12px` is the canonical card radius — used everywhere a content container needs slight rounding without feeling playful.

Scrollbar thumb uses `3px` radius — barely rounded, deliberately unobtrusive.

## Do's and Don'ts

**Do:**
- Use DM Mono exclusively for metadata, tags, labels, logotype, code, and "register-switching" contexts. Never for body copy or headings.
- Apply negative letter spacing (`-0.02em` to `-0.03em`) on all headings above `24px`.
- Keep the accent to a single role per component — highlight OR border OR fill, not all three simultaneously (exception: availability indicator, which is a special status element).
- Preserve the parchment surface distinction: `#f5f0e8` page background, `#fdfaf4` card/input surfaces.
- Maintain the two-layer shadow in light mode; remove it entirely in dark mode.
- Use `transition: background-color 0.3s ease, border-color 0.3s ease, color 0.3s ease` globally for smooth theme switching.
- Use `12px` radius for all content cards — project, blog, contact, CTA, message bubbles.
- Use black text (`#000`) on the accent fill for text buttons. Use white only for small icon-only buttons.

**Don't:**
- Don't use pure white (`#ffffff`) or pure black (`#000000`) in the palette — exception is `#000` for text on accent-fill buttons (contrast requirement).
- Don't introduce a second accent color. The burnt orange carries all interactive meaning.
- Don't use DM Sans in weight 700 or above — the system stops at 600.
- Don't apply the hero grid decoration outside of hero/full-bleed sections.
- Don't use box shadows in dark mode.
- Don't exceed `14px` border-radius on any interactive container. Cards top at `12px`, inputs at `14px`.
- Don't use Fraunces in production UI without an explicit editorial context — it's loaded but reserved.
